# 🎭 Распознавание эмоций по голосу

Дипломный проект по распознаванию и классификации эмоций в речи на русском языке.


## Что готово на 23.04.2026:
- Кардинально переработана загрузка всех датасетов. Теперь каждый датасет загружается в одинаковых условиях.
- Добавлен датасет DUSHA (от SberServices).
- Добавлен датасет CREMA_D.
- Обновлен алгоритм последовательности выполнения .py-файла обучения модели.


## Промежуточные итоги на 27.04.2026:
- Датасет Dusha помещен "в тень", поскольку замедлял разработку по причине гигантского содержимого. Еще одна причина - небольшие технические вычислительные мощности.
- Создан файл augmentation_audio.py, который содержит универсальный(еще не доделан) алгоритм аугментации данных
- Создан файл extracter_features.py, который будет содержать в себе обьединение всех признаков для обучения модели
- В файле extracter_features.py создана универсальная функция извлечения MFCC-признаков, которая при выполнении задействует все ядра процессора.


## Промежуточные итоги на 28.04.2026:
- В ходе исследований по ускорению обработки алгоритма извлечения признаков mfcc, а также алгоритма аугментации, было выяснен предел скорости обработки - 51 итерация в секунду. На данный момент это очень медленно.
- Начато изучение алгоритма LightGBM по определенным причинам. Также рассматривается вариант переехать с обработки CPU на обработку GPU. Это должно повысить скорость обработки данных.


## Промежуточные итоги на 28.04.2026:

После некоторых попыток как-то улучшить точность без отрицательных последствий, я пришел к выводу, что 54-60% - это предел точности модели на обоих алгоритмах.
Кроме того изучение модели на нескольких датасетах приводит к проблеме "Кросс-доменного обобщения". Это значит, что проблема в доменных сдвигах: различий языка, культуре выражения, акустических условиях. То есть это проблемы, которые не затрагивают конкретное распознавание эмоций.
Приходит понимание того, что нельзя сделать одну универсальную модель для разных различий языка, культуре выражения и акустических условиях.
